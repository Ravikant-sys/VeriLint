Active Rules Engine

The VeriLint ruleset conforms to standard RTL engineering best practices for synthesizable code arrays. Currently enabled detection mechanisms include:

  • Mixed Assignments

    Prevents assigning signals utilizing blocking (=) alongside non-blocking (<=) within `always` structures.

  • Inferred Latches

    Identifies combinational instances where if/else blocks are incomplete, risking the synthesis of unwanted latches.

  • Multi-Driver Anomalies

    Detects instances of solitary physical variables being simultaneously driven across discrete `always` blocks leading to unpredictable races.

  • Dead Code & Unused Ports

    Scans top modules and sub-modules to flag instantiated parameters, inputs, or physical wires failing to tie to actionable logic.
